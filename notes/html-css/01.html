# 01 【 BFC 】

# 1. FC 格式化上下文

FC (Formatting Context)

布局環境，頁面中的一塊渲染區域，有一套渲染規則，決定了元素將如何定位，以及和其他元素的關係、相互之間的作用。

每一個元素盒子都屬於 FC ，可能是區塊(block)元素，或是行內(inline)元素，但不會同時屬於兩種。

區域元素的布局屬於BFC(Block formatting context)

div、p、h1、h1 -> block level box 都是在BFC中布局的

行內元素的布局屬於IFC(Inline formatting context)

a、span、i、img -> inline level box 都是在IFC中布局的

# 2. BFC (Block formatting context)

# 1. MDN 上整理出在哪些具體的情況下會建立BFC

使用以下元素就會產生BFC

・根元素(<html>)

・浮動元素(元素的 float 不是 none)

・絕對定位元素(元素的 position 為 absolute 或 fixed)

・行內區塊元素(元素的 display 為 inline-block)

・表格單元格(元素的 display 為 table-call，HTML表格單元格默認為該值)，表格標題(元素的 display 為 table-caption，HTML表格單元格默認為該值)

・匿名表格單元格元素(元素的 display 為 table、table-row、table-row-group、table-header-group、table-footer-group(分別是HTML table、row、tbody、thead、tfoot的默認屬性) 或 inline-table)

・overflow計算值不為 visible 的區塊元素

・彈性元素(元素的 display 為 flex 或 inline-flex 元素的直接子元素)

・網格元素(元素的 display 為 grid 或 inline-grid 元素的直接子元素)

・display 為 flow-root 的元素

# 2. BFC 作用

・解決margin 摺疊(collapse)問題

・解決浮動高度塌陷問題

# 1. 排列方式

・由於BFC的規則，會讓元素盒子從頂部開始垂直一個接著一個排序，每個元素默認會依靠左邊緣

# 2. margin

・盒子與盒子垂直距離的間隔是由 margin決定

・同一個BFC中，相鄰的兩個區塊元素的margin會摺疊(collapse)

<!DOCTYPE html> <!-- HTML本身就是一個獨立的BFC -->
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <style>
        .box1{
            height: 200px;
            background-color: orange;

            margin-bottom: 30px;
        }
        .box1{
            height: 200px;
            background-color: purple;

            margin-top: 50px;
        }
    </style>
</head>
<body>
    <!-- box1 和 box2 被包覆在同一個 BFC(HTML) 中，所以margin會發生折疊 -->
    <div class="box1"></div>
    <div class="box2"></div>
</body>
</html>

# 3. 解決問題

  1. 解決margin 摺疊(collapse)問題

讓兩個 box 屬於不同的 BFC

<!DOCTYPE html> <!-- HTML本身就是一個獨立的BFC -->
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <style>
        /* overflow 設置，解決摺疊問題 */
        .container{
            overflow: auto;
        }
        .box1{
            height: 200px;
            background-color: orange;

            margin-bottom: 30px;
        }
        .box2{
            height: 200px;
            background-color: purple;

            margin-top: 50px;
        }
    </style>
</head>
<body>
    <!-- container包覆 box1，使box1屬於container的BFC-->
    <div class="container">
        <div class="box1"></div>
    </div>
    <!-- box2 屬於 HTML 的 BFC -->
    <div class="box2"></div>
</body>
</html>
  1. 解決浮動塌陷問題

float 會脫離排版，而背景沒有設置高度，高度是由子元素撐開所產生，所以看不到父元素的背景顏色

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <style>
        .container{
            background-color: orange;
        }
        .item{
            width: 400px;
            height: 200px;
            box-sizing: border-box;
            border: 1px solid #000;
            float: left;
            background-color: red;
        }
    </style>
</head>
<body>
    <div class="container">
        <div class="item"></div>
        <div class="item"></div>
        <div class="item"></div>
        <div class="item"></div>
    </div>
</body>
</html>
  1. 常見解決浮動塌陷作法：

・在父元素新增偽元素

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <style>
        .container{
            background-color: orange;
            overflow: auto;
        }
        .item{
            width: 400px;
            height: 200px;
            box-sizing: border-box;
            border: 1px solid #000;
            float: left;
            background-color: red;
        }
        .clear-fix::after {
            content: '';
            display: block;
            clear: both;
            height: 0; 
            visibility: hidden;
        }
    </style>
</head>
<body>
    <div class="container clear-fix">
        <div class="item"></div>
        <div class="item"></div>
        <div class="item"></div>
        <div class="item"></div>
    </div>
</body>
</html>
  1. ・也可以使用 BFC 解決高度塌陷問題：

提示

BFC可以解決元素高度塌陷問題，此說法並不準確

可以解決浮動(float)元素高度塌陷

無法解決絕對定位(position: absolute)元素高度塌陷

舉例：

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <style>
        .container{
            background-color: orange;
            position: relative;
        }
        .item{
            width: 400px;
            height: 200px;
            box-sizing: border-box;
            border: 1px solid #000;
            float: left;
            background-color: red;
            position: absolute;
        }
    </style>
</head>
<body>
    <div class="container clear-fix">
        <div class="item"></div>
        <div class="item"></div>
        <div class="item"></div>
        <div class="item"></div>
    </div>
</body>
</html>

BFC解決高度塌陷問題需要滿足兩個條件

1.浮動元素的父元素觸發BFC，形成獨立的區塊格式化上下文(Block formatting context)

2.浮動元素的父元素高度是auto

3.浮動元素的父元素必須有 overflow: auto or hidden or scroll，不能有visible

・如果只有inline-level，是行高的頂部和底部的距離

・如果有block-level，是盒子上邊緣和下邊緣之間的距離

(！！！重要！！！)・如果有絕對定位，將被忽略(子元素有，父元素的高度被忽略)

・如果有浮動元素，會增加高度以包覆至浮動元素的下邊緣
