# 01 【 介紹 】

# 1. 分類

# 1. 關係型資料庫 (RDMBS)

・ MySQL、Oracle、DB2、SQL Server

・ 資料結構為表格

# 2. 非關係型資料庫 (No SQL Not Only SQL)

・ MongoDB、Redis

・ 資料結構由鍵值(key-value)對組成

・ MongoDB為文件資料庫

# 2. MongoDB

# 1. 介紹

・ 為快速開發網際網路WEB應用而設計的資料庫系統

・ 設計目標為簡潔、靈活、可擴展的資料庫儲存方式

・ 資料模型是文件導向的，一種類似JSON的結構，簡單理解MongoDB資料庫存儲存的是各式各樣的JSON(BSON)

# 3. 概念

# 資料庫(databese)

資料庫是一個倉庫，在倉庫中可以存放集合

# 集合(collection)

集合類似陣列，在集合中可以存放文件

# 文件(document)

資料庫的最小單位，儲存和操作的內容都是文件

在MongoDB中，資料庫和集合都不需要手動建立，當建立文件時，如果文件所在的集合或資料庫不存在時會自動建立資料物和集合
