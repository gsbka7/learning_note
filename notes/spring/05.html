# 05 【 基於xml的Spring應用】

Spring開發中主要是對Bean的配置

Bean的常用配置：

# 1. Bean基礎配置

<bean id="userDao" class="com.spring.dao.impl.UserDaoImpl">

此時儲存到Spring容器中的Bean的beanName是userDao，值是UserDaoImpl物件，可以根據beanName取得Bean實體

applicationContext.getBean("userDao");

如果不配置id，則Spring會把目前Bean實體的權限名做為beanName

applicationContext.getBean("com.spring.dao.impl.UserDaoImpl");

# 2. Bean的別名設置

可以為Bean指定多個別名，根據別名取得Bean物件

<bean id="userDao" name="aaa, bbb" class="com.spring.dao.impl.UserDaoImpl">

此時多個名稱都可以取得UserDaoImpl實體物件

如果沒有設定id，beanName就是第一個別名

如果也沒有設定別名，beanName就是權限名

applicationContext.getBean("userDao");
applicationContext.getBean("aaa");
applicationContext.getBean("bbb");

# 3. Bean的範圍設置(scope)

預設情況下，單純的Spring環境Bean的作用範圍有兩個

。 singleton：單例，預設值，Spring容器建立的時候，會就對Bean實體化，並儲存到容器內部，每次getBean時都是從容器中取得相同的Bean物件實體

<bean id="userDao" class="com.spring.dao.impl.UserDaoImpl" scope="singleton">

Object userDao1 = applicationContext.getBean("userService");
Object userDao2 = applicationContext.getBean("userService");

// 兩個會是一樣結果
System.out.println(userDao1);
System.out.println(userDao2);

。 prototype：原型，Spring容器初始化時不會建立Bean實體，當調用getBean時才會實體化Bean，每次getBean都會建立一個新的Bean物件實體

<bean id="userDao" class="com.spring.dao.impl.UserDaoImpl" scope="prototype">

Object userDao1 = applicationContext.getBean("userService");
Object userDao2 = applicationContext.getBean("userService");

// 兩個會是不同結果
System.out.println(userDao1);
System.out.println(userDao2);

# 4. Bean的延遲加載

當lazy-init設置為true時為延遲加載，也就是當Spring容器建立的時候，不會立即建立Bean實體物件，等待用到時再建立Bean實體物件並儲存到容器中，該Bean本質上還是單例的

對BeanFactory是無效的

<bean id="userDao" class="com.spring.dao.impl.UserDaoImpl" lazy-init="true">

# 5. Bean的 初始化 與 銷毀

Bean在被實體化後，可以執行指定的初始化方法完成初始化操作

Bean在銷毀之前，可以執行指定的銷毀方法完成銷毀操作

<bean id="userService" class="com.sp.service.impl.UserServiceImpl" init-method="init" destroy-method="destroy">

package com.sp.service.impl;

import com.sp.dao.UserDao;
import com.sp.service.UserService;

public class UserServiceImpl implements UserService {

    public void init() {
        System.out.println("初始化方法");
    }

    public void destroy() {
        System.out.println("銷毀方法");
    }
}

# initializingBean接口

Bean實體物件建立後，initializingBean方法被調用，然後init-method方法之前被調用

package com.sp.service.impl;

import com.sp.dao.UserDao;
import com.sp.service.UserService;
import org.springframework.beans.factory.InitializingBean;

public class UserServiceImpl implements UserService, InitializingBean {

    public void init() {
        System.out.println("初始化方法");
    }

    public void destroy() {
        System.out.println("銷毀方法");
    }

    private UserDao userDao;

    // set方法，UserService依賴UserDao的注入
    // BeanFactory去調用該方法，從容器(BeanFactory)中取得UserDao設置到此處
    public void setUserDao(UserDao userDao) {
        System.out.println("BeanFactory去調用該方法取得UserDao設置到此處"+userDao);

        // 賦值給成員變數
        this.userDao = userDao;
    }

    @Override
    public void afterPropertiesSet() throws Exception {
        System.out.println("afterPropertiesSet方法執行");
    }
}

# 6. Bean實體化的配置

。 構造方式實體化：底層通過構造方法對Bean進行實體化

public UserServiceImpl(String name) {
    System.out.println("有參構造");
}

<bean id="userService" class="com.sp.service.impl.UserServiceImpl">
    <!--設定有參構造-->
    <constructor-arg name="name" value="value"></constructor-arg>
</bean>

。 工廠方式實體化：底層通過調用自定義的工廠方法對Bean進行實體化

# 1. 靜態工廠方法實體化Bean

package com.sp.factory;

import com.sp.dao.UserDao;
import com.sp.dao.impl.UserDaoImpl;

public class MyBeanFactory1 {
    public static UserDao userDao() {
        // 在Bean建立之前可以進行一些其他業務邏輯操作
        return new UserDaoImpl();
    }
}

使用constructor-arg設定參數

只要是最終產生Bean的方法，內部需要的參數都可以透過constructor-arg設定，這裡的constructor並代表構造函式，而是構造參數(構造Bean的方法的參數)

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

    <!--不是把MyBeanFactory1變成物件，而是去找內部的userDao方法，
    把方法的返回值當作物件，再以指定的id作為beanName儲存到容器中-->
    <bean id="userDao1" class="com.sp.factory.MyBeanFactory1" factory-method="userDao">
        <!--name：參數名-->
        <!--value：參數值-->
        <constructor-arg name="name" value="value"></constructor-arg>
        <constructor-arg name="age" value="18"></constructor-arg>
    </bean>

    <!--動態工廠調用時必須要有物件，再用物件去調用方法-->
    <!--配置工廠物件-->
    <bean id="myBeanFactory2" class="com.sp.factory.MyBeanFactory2"></bean>
    <!--userDao2源自於MyBeanFactory2工廠物件當中的userDao方法-->
    <bean id="userDao2" factory-bean="myBeanFactory2" factory-method="userDao">
        <constructor-arg name="name" value="value"></constructor-arg>
        <constructor-arg name="age" value="18"></constructor-arg>
    </bean>

</beans>

# 4. 實現FactoryBean規範延遲實體化Bean

不需要額外指定方法，具備一定的規範

把需要產生某些Bean的需求放入緩存，當要使用的時候再去產生

package com.sp.factory;

import com.sp.dao.UserDao;
import com.sp.dao.impl.UserDaoImpl;
import org.springframework.beans.factory.FactoryBean;

public class MyBeanFactory3 implements FactoryBean<UserDao> {

    // 返回的是誰的Bean
    @Override
    public UserDao getObject() throws Exception {
        return new UserDaoImpl();
    }

    // 返回的Bean的類型
    @Override
    public Class<?> getObjectType() {
        return UserDao.class;
    }
}

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

    <!--不是把MyBeanFactory1變成物件，而是去找內部的userDao方法，
    把方法的返回值當作物件，再以指定的id作為beanName儲存到容器中-->
    <bean id="userDao1" class="com.sp.factory.MyBeanFactory1" factory-method="userDao">
        <!--name：參數名-->
        <!--value：參數值-->
        <constructor-arg name="name" value="value"></constructor-arg>
        <constructor-arg name="age" value="18"></constructor-arg>
    </bean>

    <!--動態工廠調用時必須要有物件，再用物件去調用方法-->
    <!--配置工廠物件-->
    <bean id="myBeanFactory2" class="com.sp.factory.MyBeanFactory2"></bean>
    <!--userDao2源自於MyBeanFactory2工廠物件當中的userDao方法-->
    <bean id="userDao2" factory-bean="myBeanFactory2" factory-method="userDao">
        <constructor-arg name="name" value="value"></constructor-arg>
        <constructor-arg name="age" value="18"></constructor-arg>
    </bean>

    <!--根據id取得MyBeanFactory3內部getObject()方法的返回的物件(new UserDaoImpl())-->
    <bean id="userDao3" class="com.sp.factory.MyBeanFactory3"></bean>

    <bean id="userService" class="com.sp.service.impl.UserServiceImpl">
        <!--name：方法名-->
        <!--ref：要注入的bean的id-->
        <property name="userDao" ref="userDao"></property>

        <!--設定有參構造-->
        <constructor-arg name="name" value="value"></constructor-arg>
    </bean>

    <bean id="userDao" class="com.sp.dao.impl.UserDaoImpl"></bean>
</beans>

package com.sp.test;

import com.sp.service.UserService;
import org.springframework.context.ApplicationContext;
import org.springframework.context.support.ClassPathXmlApplicationContext;

public class ApplicationContextTest {
    public static void main(String[] args) {
        // 當要加載配置文件，建立容器的時候，getBean方法還沒有被調用
        // 當執行getBean()時，才去調用MyBeanFactory3裡的getObject()，才把返回的UserDaoImpl()存進緩存(FactoryBeanObjectCache)中
        // 下次再去調用userDao3這個Bean，就會直接從緩存取得

    }
}

# 7. Bean的依賴注入配置方式

# Bean的依賴注入方式主要有兩種

注入的是普通屬性，使用value

注入的是物件引用(一個Bean需要注入另一個Bean，而這個Bean也存在容器中)，使用ref

# 依賴注入的資料類型

。 普通資料類型：String、int、boolean等，通過value屬性指定

。 參考資料類型：UserDaoImpl、DataSource等，通過ref屬性指定

。 集合資料類型：List、Map、Properties等

# 範例：集合資料類型
package com.sp.service;

public interface UserService {
    public void show();
}

package com.sp.service.impl;

import com.sp.dao.UserDao;
import com.sp.service.UserService;
import org.springframework.beans.factory.InitializingBean;

import java.util.List;

public class UserServiceImpl implements UserService {

    // 1. 注入List集合
    // 1.1 注入普通字串
    private List<String> stringList;
    // 透過set方法注入
    public void setStringList(List<String> stringList) {
        this.stringList = stringList;
    }
    // 1.2 注入物件
    private List<UserDao> userDaoList;
    // 透過set方法注入
    public void setUserDaoList(List<UserDao> userDaoList) {
        this.userDaoList = userDaoList;
    }


    // 2. 注入Set集合
    // 2.1 注入普通字串
    private Set<String> strSet;
    public void setStrSet(Set<String> strSet) {
        this.strSet = strSet;
    }
    // 1.2 注入物件
    private Set<UserDao> userDaoSet;
    public void setUserDaoSet(Set<UserDao> userDaoSet) {
        this.userDaoSet = userDaoSet;
    }
    

    // 3. 注入Map集合
    private Map<String, UserDao> map;
    public void setMap(Map<String, UserDao> map) {
        this.map = map;
    }


    // 4. 注入Properties集合
    private Properties properties;
    public void setProperties(Properties properties) {
        this.properties = properties;
    }

    public void show() {
        System.out.println(stringList);
        System.out.println(userDaoList);
        System.out.println(strSet);
        System.out.println(userDaoSet);
        System.out.println(map);
        System.out.println(properties);
    }
}

            <!--內部是集合所以用list-->

                <!--內部裝了普通的字串-->
                <value>a</value>
                <value>b</value>
                <value>c</value>
            </list>
        </property>
        <property name="userDaoList">
            <list>
                <!--內部裝的是物件引用-->
                <!--每一個都代表建立一個物件-->
                <bean class="com.sp.dao.impl.UserDaoImpl"></bean>
                <bean class="com.sp.dao.impl.UserDaoImpl"></bean>
                <bean class="com.sp.dao.impl.UserDaoImpl"></bean>

                <!--透過ref引用-->
                <ref bean="userDao1"></ref>
                <ref bean="userDao2"></ref>
                <ref bean="userDao3"></ref>
            </list>
        </property>
        
        <!--Set集合-->
        <property name="strSet">
            <set>
                <value>x</value>
                <value>y</value>
                <value>z</value>

        </property>
        <!--Set集合是有序的-->
        <property name="userDaoSet">
            <set>
                <ref bean="userDao1"></ref>

            </set>
        </property>
        
        <!--Map集合-->
        <property name="map">
            <map>
            <!--map為鍵值對-->
            <!--普通字串使用key，物件使用key-ref-->
            <!--普通字串使用value，物件使用value-ref-->

            <entry key="d2" value-ref="userDao2"></entry>
            <entry key="d3" value-ref="userDao3"></entry>
            </map>
        </property>
        
        <property name="properties">
            <props>
                <!--鍵跟值都是字串-->
                <prop key="p1">p1</prop>
                <prop key="p2">p2</prop>
                <prop key="p3">p3</prop>
            </props>
        </property>
    </bean>

    <bean id="userDao2" class="com.sp.dao.impl.UserDaoImpl"></bean>
    <bean id="userDao3" class="com.sp.dao.impl.UserDaoImpl"></bean>
</beans>

# 12. Bean實體化的基本流程

Spring容器在進行初始化時，會將xml配置的<bean>的資訊封裝成一個BeanDefinition物件，所有的BeanDefinition儲存到一個名為beanDefinitionMap的Map集合中，Spring框架再對該Map進行遍歷，使用反射建立Bean實體物件，建立好的Bean物件儲存在一個名為singletonObjects的Map集合中，當調用getBean方法時則最終從該Map集合中取出Bean實體物件返回

。 加載xml配置文件，解析獲取配置中的每個<bean>的資訊，封裝成一個個的BeanDefinition物件

。 將BeanDefinition儲存在一個名為beanDefinitionMap的Map<String，BeanDefinition>中

。 ApplicationContext底層遍歷beanDefinitionMap，建立Bean實體物件

。 建立好的Bean實體物件，被儲存到一個名為singletonObjects的Map<String，Object>中

。 當執行applicationContext.getBean(beanName）時，從singletonObjects去匹配Bean實體返回

# BeanFactoryPostProcessor在SpringBean實體化過程的體現 - 1

。 Reader讀取xml配置文件

。 解析Bean標籤，把每個bean標籤都定義成beanDefinition，並存入beanDefinitionMap中

。 再從beanDefinitionMa中遍歷取出每個beanDefinition，通過反射機制，建立成bean實體物件

。 之後再儲存到singletonObjects中

。 getBean()是從singletonObjects中取出

# 13. Bean後處理器

Spring的後處理器是Spring對外開發的重要擴展點，允許我們介入到Bean的整個實體化流程中來，以達到動態註冊BeanDefinition，動態修改BeanDefinition，以及動態修改Bean的作用。Spring主要有兩種後處理器：

。 BeanFactoryPostProcessor：Bean工廠後處理器，在BeanDefinitionMap填充完畢，Bean實體化之前執行

。 BeanPostProcessor：Bean後處理器，一般在Bean實體化之後，填充到單例池singletonObjects之前執行。

# 1. Bean工廠後處理器 BeanFactoryPostProcessor

BeanFactoryPostProcessor是一個接口規範，實現了該接口的類只要交由Spring容器管理的話，那麼Spring就會回調該接口的方法，用於對BeanDefinition註冊和修改的功能

# 定義

public interface BeanFactoryPostProcessor {
    void postProcessBeanFactory(ConfigurableListableBeanFactory var1) throws BeansException;
}

       http://www.springframework.org/schema/beans/spring-beans.xsd">

        <!--不需要取id，時機到了就會自動調用-->

public class MyBeanPostProcessor implements BeanFactoryPostProcessor {
    // 參數其實就是BeanFactory
    // 交給Spring管理：建立接口 -> 重寫方法 -> 配置xml文件
    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) throws BeansException {
        System.out.println("BeanDefinition填充完畢後回掉該方法");

        // 1. 修改某個BeanDefinition
        BeanDefinition beanDefinition1 = beanFactory.getBeanDefinition("userService");
        // 修改權限名，取得的結果從UserServiceDao變成UserDao物件
        beanDefinition1.setBeanClassName("com.sp.dao.impl.UserDaoImpl");

        // 2. 動態註冊BeanDefinition
        BeanDefinition beanDefinition2 = new RootBeanDefinition();
        beanDefinition2.setBeanClassName("com.sp.dao.impl.PersonDaoImpl");
        // DefaultListableBeanFactory為beanFactory的父類，強轉後使用內部的方法
        DefaultListableBeanFactory defaultListableBeanFactory = (DefaultListableBeanFactory) beanFactory;
        defaultListableBeanFactory.registerBeanDefinition("personDao", beanDefinition2);
    }
}

# 3. Bean後處理器 BeanPostProcessor

Bean被實體化後，到最終緩存到名為singletonObjects單例池之前，中間會經過Bean的初始化過程，例如：屬性的填充、初始方法init的執行等，其中有一個對外進行擴展的點BeanPostProcessor，我們稱為Bean後處理。跟上面的Bean工廠後處理器相似，它也是一個接口，實現了該接口並被容器管理的BeanPostProcessor，會在流程節點上被Spring自動調用。

package com.sp.processor;

import org.springframework.beans.BeansException;
import org.springframework.beans.factory.config.BeanPostProcessor;

public class MyBeanPostProcessor implements BeanPostProcessor {
    // 兩個方法的註解為@Nullable，代表可以不實現

    // 流程
    // 1. 實體化物件
    // 2. 調用before方法
    // 3. 執行物件初始化方法
    // 4. 調用after方法

    // Bean物件建立完畢後先執行
    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {
        System.out.println(beanName + ":postProcessBeforeInitialization");
        return bean;
    }

    // 後執行
    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
        System.out.println(beanName + ":postProcessAfterInitialization");
        return bean;
    }
}

package com.sp.processor;

import org.springframework.beans.BeansException;
import org.springframework.beans.factory.config.BeanPostProcessor;

import java.lang.reflect.Proxy;
import java.util.Date;

// Bean物件建立完畢後執行
public class TimeLogBeanPostProcessor implements BeanPostProcessor {
    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
        // 使用動態代理對目標Bean進行增強，返回proxy物件，進而儲存到單例池singletonObjects中
        Object beanProxy = Proxy.newProxyInstance(
                // 加載器
                bean.getClass().getClassLoader(),
                //
                bean.getClass().getInterfaces(),
                (proxy, method, args) -> {
                    // 1. 輸出開始時間
                    System.out.println("方法:" + method.getName() + "-開始時間:" + new Date());
                    // 2. 執行目標方法
                    Object result = method.invoke(bean, args);
                    // 3. 輸出結束時間
                    System.out.println("方法:" + method.getName() + "-結束時間:" + new Date());
                    return result;
                }
        );
        return beanProxy;
    }
}

# 14. Spring的生命週期

SpringBean的生命周期是從Bean實體化之後，即通過反射創建出物件之後，到Bean成為一個完整物件，最終儲存到單例池(singletonObjects)中，這個過程被稱為SpringBean的生命週期。

# 1. 生命週期的執行步驟

# Spring Bean的生命周期大體上分為三個階段：

。 Bean的實體化階段：Spring框架會取出BeanDefinition的資訊進行判斷目前Bean的範圍是否是singleton的，是否不是延退加載的，是否不是FactoryBean等，最終將一個普通的singleton的Bean通過反射進行實體化；

。 Bean的初始化階段：Bean創建之後還僅僅是個"半成品”，還需要對Bean實體的屬性進行填充、執行一些Aware接口方法、執行BeanPostProcessor方法、執行InitializingBean接口的初始化方法、執行自定義初始化init方法等。

。 Bean的完成階段：經過初始化階段，EBean就成為了一個完整的SpringBean，被存儲到單例池singletonObjects中去了，即完成了SpringBean的整個生命周期

# 2. SpringBean的初始化過程

。 Bean實體的屬性填充

。 Aware接口屬性注入

。 BeanPostProcessor的before()方法回調

。 InitializingBean接口的初始化方法回調

。 自定義初始化方法init回調

。 BeanPostProcessor的after()方法回調

# Bean實體的屬性填充

Spring在進行屬性注入時，會分為如下幾種情況：

。 注入普通屬性，String、int或存儲基本類型的集合時，直接通過set方法的反射設置進去

。 注入單向物件引用屬性時，從容器中getBean獲取後通過set方法反射設置進去，如果容器中沒有，則先創建被注入物件Bean實體（完成整個生命週期）後，再進行注入操作

。 注入雙向物件引用屬性時，涉及了循環引用（循環依賴）問題

多個實體之間相互依賴並形成閉環的情況就叫做「循環依賴」，也叫做「循環引用」

# 3. Aware接口

AWare接口是一種框架輔助屬性注入的一種思想，其他框架中也可以看到類似的接口。框架具備高度封裝性，我們接觸到的一般都是業務程式碼，一些底層功能API不能輕易的獲取到，但是這不意味著永遠用不到這些物件，如果用到了，就可以使用框架提供的類似Aware的接口，讓框架給我們注入該物件

常用的Aware接口：
