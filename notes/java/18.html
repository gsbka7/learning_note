# 18 【IO流】

儲存和讀取資料的解決方案

# 1. 流的分類

# 總結

  1. IO流是儲存和讀取資料的解決方案

I：input，O：output

流：像水流一樣傳輸資料

  1. 作用

用於讀寫資料(本地檔案、位元組輸出流網路)

  1. 流向分類

輸出流：程式 -> 檔案

輸入流：檔案 -> 程式

  1. 操作文件的類型分類

位元組流：可以操作所有類型的檔案

字元流：只能操作純文本檔案(可以用記事本打開並可以讀懂的檔案)，例如：txt、md、xml、lrc

# 3. 位元組流

# 1. FileOutputStream

操作本地檔案的位元組輸出流，可以將程式中的資料寫入本地檔案中

# 1. 書寫步驟：

。建立位元組輸出流物件

參數是字串表示的路徑或是File物件

如果檔案不存在，會建立新的檔案，但是要保證父級路徑是存在的

如果檔案已經存在，會清空內容，並寫上新的內容

。寫入資料

write方法的參數是整數，實際上寫入的是ASCII表中對應的字元

import java.io.FileOutputStream;

# 2. 寫入資料的三種方式

import java.io.FileOutputStream;
import java.io.IOException;

public class ByteStreamDemo {

        // 1. 建立物件
        FileOutputStream fos = new FileOutputStream("C:\\java\\a.txt");

        // 2. 寫入資料

        // void write(byte[] b, int off, int len)

import java.io.FileOutputStream;
import java.io.IOException;

}

# 2. FileInputStream

操作本地檔案的字元輸入流可以把本地檔案中的資料讀取到程式中

# 1 .書寫步驟：

。建立字元輸入流物件

如果程式不存在，直接報錯

。讀取資料

read()方法讀取一次就移動一次指針

書寫一次只讀取一個字元，讀取出的是ASCII對應的數字

如果讀取到檔案的末尾，已經沒有資料，read方法返回-1

空格也是資料，ASCII是32

。釋放資源

每次使用完流之後都要釋放資源，否則會佔用記憶體

import java.io.FileInputStream;

        FileInputStream fis = new FileInputStream("C:\\java\\a.txt");

        int b1 = fis.read();
        System.out.println((char)b1);

        // 3. 釋放資源
        fis.close();
    }
}

        // 3. 釋放資源

# 4. 文件拷貝(一次讀寫多個字元)
import java.io.FileInputStream;
import java.io.FileOutputStream;
import java.io.IOException;

public class ByteStreamDemo {
    public static void main(String[] args) throws IOException {
        long start = System.currentTimeMillis(); // 開始時間

        // 1. 建立物件
        FileInputStream fis = new FileInputStream("C:\\java\\a.mp4");
        FileOutputStream fos = new FileOutputStream("C:\\java\\copy.mp4");
        // 2. 拷貝
        int len;
        byte[] bytes = new byte[1024 * 1024 * 5];
        while((len = fis.read(bytes)) != -1) {
            // 不要將所有資料都寫目的地，因為最後一次讀取時可能裝不滿
            // 裝多少，寫入多少
            fos.write(bytes, 0, len);
        }

        // 3. 釋放資源
        fos.close();
        fis.close();

        long end = System.currentTimeMillis(); //　結束時間

        System.out.println(end - start);
    }
}

# 基本作法

import java.io.FileInputStream;
import java.io.FileOutputStream;
import java.io.IOException;

public class ByteStreamDemo2 {
    public static void main(String[] args) throws IOException {
        FileInputStream fis = null;
        FileOutputStream fos  = null;

        try {
            // 1. 建立物件
            fis = new FileInputStream("C:\\java\\a.mp4");
            fos = new FileOutputStream("C:\\java\\copy.mp4");
            // 2. 拷貝
            int len;
            byte[] bytes = new byte[1024 * 1024 * 5];
            while((len = fis.read(bytes)) != -1) {
                // 不要將所有資料都寫目的地，因為最後一次讀取時可能裝不滿
                // 裝多少，寫入多少
                fos.write(bytes, 0, len);
            }

        } catch (IOException e) {
            throw new RuntimeException(e);
        } finally {
            // 3. 釋放資源
            if(fos != null) {
                try{
                    fos.close();
                }catch(IOException e) {
                    e.printStackTrace();
                }
            }

            if(fis != null) {
                try{
                    fis.close();
                }catch(IOException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

# 2. 亂碼

原因一：讀取資料時未讀取完整個漢字

原因二：編碼和編碼時的方式不統一

# 如何不產生亂碼

  1. 不要用位元組流讀取文本(每一種文字的編碼的位元組長度不同)

  2. 編碼解碼時使用同一個碼表，同一個編碼方式

# 3. Java實現 編碼、解碼

import java.io.UnsupportedEncodingException;
import java.util.Arrays;

public class CharStreamDemo {
    public static void main(String[] args) throws UnsupportedEncodingException {
        // 編碼
        // 1. 預設編碼
        String str = "aaa一二三";
        byte[] bytes1 = str.getBytes();
        System.out.println(Arrays.toString(bytes1));

        // 2. BIG5 // 一個英文一個位元組，一個中文兩個位元組
        byte[] bytes2 = str.getBytes("BIG5");
        System.out.println(Arrays.toString(bytes2));

        // 解碼
        String str2 = new String(bytes1);
        System.out.println(str2);

        // 編碼時使用UTF8，解碼時使用BIG5，出現亂碼
        String str3 = new String(bytes1, "BIG5");
        System.out.println(str3);
    }
}

# 空參read方法

。 字元輸入流的底層就是位元組流，也是一個字元一個字元讀取的

。 如果遇到中文就會一次讀取多個

。 在讀取之前，方法的底層還會進解碼並轉成十進制

import java.io.FileReader;
import java.io.IOException;

public class CharStreamDemo {
    public static void main(String[] args) throws IOException {
        // 1. 建立物件並關聯本地檔案
        FileReader fr = new FileReader("C:\\java\\a.txt");

        // 2. 讀取資料 read()

        while ((ch = fr.read()) != -1) {

        }

    }
}

import java.io.FileReader;

import java.io.*;

public class BufferedStreamDemo {
    public static void main(String[] args) throws IOException {
        // 建立物件
        // 參數：8192緩衝區，或手動設定大小

        BufferedOutputStream bos = new BufferedOutputStream(new FileOutputStream("C:\\java\\copy.txt"));

        // 拷貝
        byte[] bytes = new byte[1024];
        int len;
        while ((len = bis.read(bytes)) != -1) {
            bos.write(bytes, 0, len);
        }

        // 釋放資源
        bos.close();
        bis.close();
    }
}

# 3. 總結

  1. 緩衝流的種類

。位元組緩衝輸入流：BufferedInputStream

。位元組緩衝輸出流：BufferedOutputStream

。字元緩衝輸入流：BufferedReader

。字元緩衝輸出流：BufferedWriter

  1. 提高效能原理

。緩衝流自帶了長度8192的緩衝區

位元組緩衝流的緩衝區：byte類型，長度8K

字元緩衝流的緩衝區：char類型，長度16K

。可以顯著提高位元組流的讀寫性能

。對於字元流提升不明顯，關鍵是有兩個特有方法

  1. 字元緩衝流的特有方法

字元緩衝輸入流BufferedReader：readLine()：一次讀取一整行資料

字元緩衝輸出流BufferedWriter：newLine()：跨平台換行

# 4. 範例

先是創建了一個代理的對象star,然後star需要三個參數

第一個參數是類加載器用來加載對象的,第二個參數是獲得star的所有方法,第三個是指定star要做啥

然後第三個需要調用invoke方法,而invoke方法中又需要三個參數,第一個是代理的對象也就是star

package dynamicproxy;

public class Test {
    public static void main(String[] args) { // 相當於外部
        /*
         * 需求：外部的人想要Star唱一首歌
         * 1. 取得代理的物件
         * 代理物件(返回值) = ProxyUtil.createProxy(Star的物件)
         * 2. 調用代理的唱歌方法
         * 代理物件.唱歌方法("歌曲名稱")
         * // 當外界調用唱歌方法後，就會調用下面的invoke方法，因為invoke方法裡就是代理要做的事情
         */

        // 1. 取得代理的物件
        BigStar bigStar = new BigStar("明星");
        Star proxy = ProxyUtil.createProxy(bigStar); // 表示給bigStar產生代理，方法的結果就是代理的物件

        // 2. 調用唱歌的方法
        String result = proxy.sing("歌曲名稱");
        System.out.println(result);

        // 2. 調用跳舞的方法
        proxy.dance(); // 無參，無返回值
    }
}

package dynamicproxy;

import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.lang.reflect.Proxy;

public class ProxyUtil {

    /*
    * 需求：外部的人想要Star唱一首歌
    * 1. 取得代理的物件
    * 代理物件(返回值) = ProxyUtil.createProxy(Star的物件)
    * 2. 調用代理的唱歌方法
    * 代理物件.唱歌方法("歌曲名稱")
    * // 當外界調用唱歌方法後，就會調用下面的invoke方法，因為invoke方法裡就是代理要做的事情
     */


    // 給BigStar物件，建立一個代理
    // 形參：被代理的BigStar物件
    // 返回值：給Star建立的代理。代理要實現Star接口，所以寫Star
    public static Star createProxy(BigStar bigStar) {
        // 返回的是代理的物件
        // 參數一：用於指定哪個類加載器(在執行時把位元組碼文件加載到記憶體中)，去加載產生的代理類
        // 參數二：指定接口，這些接口用於指定產生的代理有什麼方法
        // 參數三：用來指定產生的代理物件要做什麼
        Star star = (Star)Proxy.newProxyInstance(ProxyUtil.class.getClassLoader(), // 找到是誰把目前這個類加載到內存，並且讓這個類再去加載目前的代理
                new Class[]{Star.class}, // 把接口的位元組碼放在接口中 // 表示產生的這個代理可以代理Start這個接口裡所有的方法
                new InvocationHandler() { // 報錯是因為返回值是Object，但用Start接收，就會報錯。需要將Star類型強轉
                    @Override
                    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
                        // 參數一：目前代理的物件
                        // 參數二：要執行的方法 sing
                        // 參數三：調用sing方法時，傳遞的實參
                        if("sing".equals((method.getName()))) { // 如果是調用的是唱歌方法
                            System.out.println("準備麥克風"); // 代理要做的事
                        }else if("dance".equals(method.getName())) {
                            System.out.println("準備場地");
                        }

                        // 去找BigStar唱歌或跳舞
                        // 調用BigStar裡唱歌或跳舞的方法
                        // 反射，用bigStar調用，參數args，有返回值就進行return
                        return method.invoke(bigStar, args);
                    }
                }
        ); // 通過InvocationHandler接口來實現

        return star;
    }
}

package dynamicproxy;

public class BigStar implements Star{
    private String name;

    public BigStar() {
    }

    public BigStar(String name) {
        this.name = name;
    }

    // 唱歌
    @Override
    public String sing(String name) {
        System.out.println(this.name + "唱" + name);
        return "謝謝";
    }

    // 跳舞
    @Override
    public void dance() {
        System.out.println(this.name + "正在跳舞");
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    @Override
    public String toString() {
        return "BigStar{" +
                "name='" + name + '\'' +
                '}';
    }
}

package dynamicproxy;

public interface Star {
    // 把所有想要被代理的方法定義在接口中

    public abstract String sing(String name);

    // 跳舞
    public abstract void dance();
}
