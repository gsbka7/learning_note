# 06 【物件導向 進階】

# 1. static

static表示靜態，是Java中的一個修飾符，可以修飾成員方法，成員變數

# 1. 成員變數：

# 特點：

被該類所有物件共享

靜態屬性不屬於物件，屬於類

隨著類的加載而加載，優先於物件存在

# 調用方式：

類名(推薦)

物件名

# 2. 成員方法

# 特點：

多用在測試類或工具類中

Javabean類中很少會用

# 調用方式：

類名(推薦)

物件名

# 3. 補充 -- 工具類

# 1. 類的差別

# Javabean類：

用來描述一類事物的類：例如Student、Teacher、Dog、Cat

# 測試類：

用來檢查其他類是否書寫正確，帶有main方法的類，是程式的入口

# 工具類：

不描述任何事物，而是幫我們做一些事情的類

  1. 類名見名知意

  2. 私有化構造

因為工具類不描述任何事物，建立物件沒有意義

# 自定義工具類範例

陣列的工具類ArrayUtil

提供一個工具方法printArr，用於返回陣列的內容。返回格式如：[1-, 20, 30, 40, 50](只考慮整數，且是一維陣列)

提供一個工具方法getAerage，用於返回平均值(只考慮浮點型陣列，且是一維陣列)

定義一個測試類TestDemo，調用該工具類的工具方法，且返回結果

public class ArrayUtil {
    private ArrayUtil() {}

    public static String printArr(int[] arr) {
        StringBuilder sb = new StringBuilder();
        sb.append("[");
        for (int i = 0; i < arr.length; i++) {
            if(i == arr.length - 1) {
                sb.append(arr[i]);
            }else{
                sb.append(arr[i] + ", ");
            }
        }
        sb.append("]");
        return sb.toString();
    }

    public static double getAverage(double[] arr) {
        double sum = 0;
        for (int i = 0; i < arr.length; i++) {
            sum = sum + arr[i];
        }
        return sum / arr.length;
    }
}

public class TestDemo {
    public static void main(String[] args) {
        int[] arr1 = {1,2,3,4,5};
        String str = ArrayUtil.printArr(arr1);
        System.out.println(str);

        double[] arr2 = {1.5,3.7,4.9,5.8,6.6};
        double avg = ArrayUtil.getAverage(arr2);
        System.out.println(avg);
    }
}

# 4. 注意點

靜態方法只能訪問靜態變數和靜態方法

非靜態方法可以訪問靜態變數或者靜態方法，也可以訪問非靜態的成員變數和非靜態的成員方法

靜態方法中沒有this關鍵字

總結

靜態方法中，只能訪問靜態

非靜態方法可以訪問所有

靜態方法中沒有this關鍵字

# 2. 繼承

# 1. 概述

Java中提供一個關鍵字extends，用於讓一個類和另一個類建立起繼承關係

public class Student extends Person() {};

Student稱為子類，Person稱為父類

# 優點：

可以把多個子類中重複的程式碼抽取到父類中，提高複用性

子類可以在父類的基礎上，增加其他的功能，使子類更強大

# 使用時機：

當類與類之間，存在相同的內容，並滿足子類是父類的一種，就可以考慮使用繼承

# 小節：

繼承是物件導向的三大特徵之一，可以讓類與類之間產生子父的關係

可以把多個子類中重複的程式碼抽取至父類中，子類可以直接使用，減少程式碼量，提高複用性

子類可以得到父類的屬性和行為，並且使用

子類可以在父類的基礎上新增其他功能

# 2. 特點

  1. Java只支持單繼承，不支持多繼承，但支持多層繼承

  2. 多層繼承：子類A繼承父類B，父類B可以繼承父類C

  3. 每一個類都直接或間接繼承Object

  4. 子類只能訪問父類中非私有(不是private)的成員

# 3. 子類能夠繼承父類中的那些內容

  1. 構造方法

非私有 跟 private 都不能繼承，當然也不能被使用

  1. 成員方法

只有父類中的虛方法才能被子類繼承

Java會在最頂級的父類設置虛方法表，把經常使用的方法放到虛方法表中，然後再交給子類

虛方法：非private、非static、非final

例如：A繼承B，B繼承C

父類C設置虛方法表交給子類B，子類B在父類C的基礎上在增加自己類中的虛方法

然後再交給子類A，子類A也會在B的基礎上增加自己類中的虛方法

public class Test {
    public static void main(String[] args) {
        Son son = new Son;
        System.out.println(son);
        
        son.FaShow1();
        son.FaShow2(); // 在父類設置為private，無法被使用
        
        son.SonShow();
    }
}

class Father{
    public void FaShow1() {
        System.out.println("public --- FaShow1");
    }
    private void FaShow2() {
        System.out.println("private --- FaShow2");
    }
}

class Son extends Father{
    public void SonShow() {
        System.out.println("public --- SonShow1");
    }
}

就近原則

    }

    String name = "Father";
}

    public void SonShow() {
        String name = "SonShow";
        System.out.println(name);

    }

public class Test {
    public static void main(String[] args) {
        Son son = new Son();
        son.SonShow();

    }
}

class Father{
    String name = "Father";
}

class Son extends Father{
    String name = "Son";

    public void SonShow() {
        String name = "SonShow";
        System.out.println(name); // SonShow // 就近原則
        System.out.println(this.name); // 在這個類的成員變數尋找，沒有就再往外尋找 // Son
        System.out.println(super.name); // Father // 子類調用父類的必涮
    }
}

# 5. 成員方法的訪問特點

this：就近原則

super：往父類尋找

package demo01;

public class Test {
    public static void main(String[] args) {
        Student stu = new Student();
        stu.lunch();

        Oversea o = new Oversea();
        o.Oversea();

    }
}

class Person{
     public void eat() {
         System.out.println("吃飯");
     }
    public void drink() {
        System.out.println("喝水");
    }

}

class Oversea extends Person{
    public void Oversea() {
        // 就近原則
        this.eat(); // Eat
        this.drink(); // Drink

        // 到父類尋找
        super.eat(); // 吃飯
        super.drink(); // 喝水
    }

    @Override
    public void eat() {
        System.out.println("Eat");
    }
    @Override
    public void drink() {
        System.out.println("Drink");
    }
}

class Student extends Person{
    public void lunch() {
        // 會有隱藏的this，會從本類中依照就近原則往父類尋找
        eat();
        drink();

        // 子類直接調用父類
        super.eat();
        super.drink();
    }
}

# 6. 方法的重寫

當父類的方法不能滿足子類現在的需求時，需要進行方法重寫

# 格式

在繼承體系中，子類出現了和父類一模一樣的方法宣告，稱子類這個方法是重寫的方法

@Override

  1. @Override是放在重寫後的方法上，校驗子類重寫時語法是否正確

  2. 加上後如果有紅色波浪線，表示語法錯誤

  3. 建議重寫方法都加上@Override

# 注意事項

  1. 重寫方法的名稱、形參列表必須與父類中的一致

  2. 子類重寫父類方法時，訪問權限子類必須大於等於父類

  3. 子類重寫父類方法時，返回值類型子類必須小於等於父類

  4. 建議：重寫的方法盡量和父類保持一致

  5. 私有方法不能被重寫

  6. 子類不能重寫父類的靜態方法，會出現錯誤

  7. 5跟6無法被加入虛方法，只有被加入虛方法比中的方法才能被重寫

# 7. 構造方法的訪問特點

・ 父類中的構造方法不會被子類繼承(違反子類名與構造方法名規則)

・ 子類中所有的構造方法默認先訪問父類中的無參構造，再執行自己

子類在初始化的時候，有可能會使用到父類中的資料，如果父類沒有完成初始化(父類的無參構造會將資料初始化)，子類將無法使用父類的資料

子類初始化之前，一定要調用父類構造方法先完成父類資料空間的初始化

・ 調用父類方法

子類構造方法的第一行語句默認都是super()，即使不寫也會存在，而且必須在第一行

如果想要調用父類有參構造，必須手動寫super()進行調用

package demo01;

public class Test {
    public static void main(String[] args) {
        // 會先調用父類無參構造再調用子類無參構造
        Student s1 = new Student();

        // 如果想要加上參數，必須在子類中寫上對應的有參構造
        Student s2 = new Student("Stu", 20);
        System.out.println(s2.name + "  "+ s2.age);

    }
}

public class Person {
    String name;
    int age;
    public Person() {
        System.out.println("父類無參構造");
    }

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

package demo01;

public class Student extends Person{
    public Student() {
        // 子類的無參構造中，默認帶有super方法，訪問父類的無參構造
        super();
        System.out.println("子類的無參構造");
    }

    public Student(String name, int age) {
        // 接收Test傳來的參數，然後再傳遞給Person的有參構造，之後再賦值給成員變數
        super(name, age);
    }
}

# 8. this、super使用總結

# this：理解為一個變數，表示目前方法調用者的地址值

# super：代表父類的儲存空間

關鍵字訪問成員變數訪問成員方法訪問構造方法
thisthis.成員變數：訪問本類成員變數this.成員方法()：訪問本類成員方法*this()：訪問本類構造方法
superthis.成員變數：訪問父類成員變數this.成員方法()：訪問父類成員方法this()：訪問父類構造方法

# this()：訪問本類構造方法

public class Test {
    public static void main(String[] args) {
        Student s = new Student();
    }
}

public class Student {
    String name;
    int age;
    String School;

    public Student() {
        // 表示調用本類其他構造方法
        // 虛擬機不會再添加super()

        this(null, 0 , "School");
        System.out.println();
    }

    // 這裡的構造方法默認的第一行會有個隱藏的super()
    // 在子類的方法中只需要有一個super去訪問父類即可
    public Student(String name, int age, String school) {
        this.name = name;
        this.age = age;
        School = school;
    }
}

# 3. 多態

多態是物件的多種形態

同類型的物件，表現出不同的形態

# 格式

父類類型 物件名稱 = 子類物件;

Father f = new Son();

# 前提：

有繼承關係

有父類參考指向子類物件：Father f = new Son();

有方法重寫

# 優點：

使用父類作為參數，可以接收所有子類的物件

體現擴展性與便利性

# 範例

public class Test {
    public static void main(String[] args) {
        Student s = new Student();
        s.setName("學生");
        s.setAge(18);

        Teacher t = new Teacher();
        t.setName("老師");
        t.setAge(30);

        Admin a = new Admin();
        a.setName("管理員");
        a.setAge(40);

        register(s);
        register(t);
        register(a);
    }

    // 這個方法可以接收老師，也能接收學生，還能接收管理員
    // 把參數寫成這三個類的父類
    public static void register(Person p) {
        p.show();
    }
}

public class Person {
    private String name;
    private int age;

    public Person() {

    }
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    public void show() {
        System.out.println(name + ", " + age);
    }
}

public class Student extends Person{
    @Override
    public void show() {
        System.out.println("學生資料：" + getName() + ", " + getAge());
    }
}

public class Teacher extends Person{
    @Override
    public void show() {
        System.out.println("老師資料：" + getName() + ", " + getAge());
    }

常數一般做為系統的配置訊息，方便維護，提高可讀性

# 命名規範：

單個單詞：全部大寫

# 細節：

final修飾的常數如果是基本類型：那麼變數儲存的資料值不能發生改變

public class Test {
    public static void main(String[] args) {
        final double PI = 3.14;
        PI = 0; // 不能發生改變
    }
}

final修飾的常數如果是參考類型：那麼變數儲存的地址值不能發生改變，物件內部屬性可以改變

public class Test {
    public static void main(String[] args) {
        final double PI = 3.14;

        final Student s = new Student("Name1", 20);

        // 地址值不能改變
        s = new Student();
        // 內部屬性可以改變
        s.setName("Name2");
        s.setAge(40);


        // 陣列
        final int[] arr = {1,2,3,4,5};
        // 地址值不能改變
        arr = new int[10];
        // 內部屬性可以改變
        arr[0] = 10;
        arr[1] = 20;

        for (int i = 0; i < arr.length; i++) {
            System.out.println(arr[i]);
        }
    }
}

將共性的行為(方法)抽取到父類後，由於每一個子類執行的內容是不同的，所以在父類中不能去定具體的方法體。該方法就可以定義為抽象方法

# 抽象類：

如果一個類中存在抽象方法，那麼該類就必須宣告為抽象類

# 作用：

抽取共性時，無法確定方法體，就把方法定義為抽象的

強制讓子類按照某種格式重寫

抽象方法所在的類，必須是抽象類

# 7. 接口中新增的方法

JDK7以前：接口中只能定義抽象方法

JDK8新特性：接口中可以定義有方法體的方法(默認、靜態)

JDK9新特性：接口中可以定義私有方法

私有方法分為兩種，普通的私有方法，靜態的私有方法

# 1. JDK8新增的默認方法

・ 允許在接口中定義默認方法，需要使用關鍵字default修飾

作用：解決接口升級的問題

・ 接口中默認方法的定義格式

public default 返回值類型 方法(參數){}

public default void show(){}

・ 接口中默認方法的注意點

  1. 默認方法不是抽象方法，所以不強制重寫。但是如果被重寫，重寫的時候去掉default關鍵字

  2. public可以省略，default不能省略

  3. 如果實現了多個接口，多接口中存在相同名字的默認方法，子類就必須對該方法進行重寫

public class Test {
    public static void main(String[] args) {
        InterImpl ii = new InterImpl();
        ii.method(); // 抽象類重寫的方法
        ii.show(); // 接口的默認方法
    }
}

public class InterImpl implements InterA, InterB{

    // 抽象方法不重寫會出現錯誤
    @Override
    public void method() {
        System.out.println("實現類重寫的抽象方法");
    }

    // 默認方法即使不重寫也不會出錯，但是如果被重寫，需要去掉default關鍵字
    @Override
    public void show() {
        System.out.println("重寫接口中的默認方法");
    }
}

public interface InterA {

    // 抽象方法
    public abstract void method();

    // 默認方法不是抽象方法，所以不強制被重寫。但是如果被重寫，需要去掉default關鍵字
    // public可以省略，default不能省略，否則會被當作抽象方法
    public default void show() {
        System.out.println("InterA接口中的默認方法 --- show");
    }
}

public interface InterB {

    public default void show() {
        System.out.println("InterB接口中的默認方法 --- show");
    }
}

# 2. JDK8新增的靜態方法

・ 定義格式

・ 接口中靜態方法的注意事項

靜態方法只能通過接口調用，不能通過實現類名或者物件名調用

public可以省略，static不能省略

public class Test {

public class InterImpl implements Inter{

    //
    @Override

# 8. 接口總結

  1. 接口代表規則，是行為的抽象。想要讓哪個類擁有一個行為，就讓這個類實現對應的接口

  2. 當一個方法的參數是接口時，可以傳遞接口所有的實現類物件，這種方式稱為接口多態

# 9. 轉接器設計模式

・ 設計模式(Design pattern)是一套被反覆使用、多數人知曉、經過分類編目的、程式碼設計經驗的總結。使用設計模式是為了可重用程式碼，讓程式碼更容易被他人理解、保證程式碼可靠性、程式的重用性

・ 轉接器設計模式：解決接口與接口實現類之間的矛盾問題

public class InterImpl extends InterAdapter{
    // 繼承轉接器
    // 要用哪個方法，就寫哪個方法

    @Override
    public void method1() {
        System.out.println("只要第一個方法");
    }
}

public interface Inter {
    // 接口中的所有方法
    public abstract void method1();
    public abstract void method2();
    public abstract void method3();
    public abstract void method4();
    public abstract void method5();
}

// 將接口中所有方法實現在轉接器中
public class InterAdapter implements Inter{
    @Override
    public void method1() {}

    @Override
    public void method2() {}

    @Override
    public void method3() {}

    @Override
    public void method4() {}

    @Override
    public void method5() {}
}

# 總結

  1. 當一個接口中抽象方法過多，但如果只想要使用一部份，就可以使用轉接器設計模式

  2. 書寫步驟

編寫中間類xxxAdapter，實現對應的接口，對接口中的抽象方法進行空實現，讓真正的實現類繼承中間類，並重寫需要用的方法

為了避免其他類建立轉接器類的物件，中間的轉接器類要用abstract進行修飾

整體可以理解為接口的實現類物件

public interface Swim {
    public abstract void swim();
}

public class Test {
    public static void main(String[] args) {
        // 接口多態
        Swim s = new Swim() { // 接口 = 實現類物件
            @Override
            public void swim() {
                System.out.println("重寫之後游泳方法");
            }
        };

        // 編譯看左邊，執行看右邊

# 使用場景

當方法的參數是接口或類的時候，以接口為例，可以傳遞這個接口的實現類物件

如果實現類只使用一次，就可以匿名內部類簡化程式碼

public abstract class Animal {
    public abstract void eat();
}

public class Dog extends Animal{
    @Override
    public void eat() {
        System.out.println("狗吃骨頭");
    }
}

public class Test {
    public static void main(String[] args) {
        // 在測試類中調用method方法
        // 寫一個子類繼承Animal類
        // 建立子類的物件，傳遞給mehtod方法
        Dog d = new Dog();
        method(d);

        // 如果Dog類只需要用一次，單獨定義一個類就太麻煩了
        method(
                new Animal() {
                    @Override
                    public void eat() {
                        System.out.println("狗吃骨頭");
                    }
                }
        );
    }

    public static void method(Animal a) { // Animal a = 子類物件：多態
        a.eat(); // 編譯看左邊，執行看右邊
    }
}
