# 19 【多線程】

# 1. 介紹

。線程：操作系統能夠進行運算調度的最小單位。被包含在進程之中，是進程的實際運作單位

應用程式當中互相獨立，可以同時執行的功能

當這樣的功能比較多，就形成了多線程

。進程：是程式的基本執行實體

# 應用場景

軟體中的耗時操作：拷貝、加載大量檔案

聊天軟體

後台伺服器

# 總結

  1. 多線程能讓程式同時處理多件事情

  2. 作用：提高效率

  3. 想要讓多件事情同時執行就需要用到多線程

# 2. 平行 與 並行

平行(concurrency：在同一時刻，有多個指令在單個CPU上交替執行

並行(parallelism)：在同一時刻，有多個指令在多個CPU上同時執行

  1. 利用Callable接口和Future接口方式實現

可以取得多線程執行的結果

import java.util.concurrent.ExecutionException;
import java.util.concurrent.FutureTask;

public class ThreadDemo {
    public static void main(String[] args) throws ExecutionException, InterruptedException {
        // 1.建立Mycallable類實現Callable接口
        // 2.重寫call(有返回值，表示多線程執行的結果)

        // 3.建立Mycallable物件(表示多線程要執行任務)
        MyCallable mc = new MyCallable();

        // 4.建立FutureTask物件(管理多線程執行的結果)
        // 泛型：多線程執行的結果
        FutureTask<Integer> ft = new FutureTask<>(mc);

        // 5.建立Thread物件，並啟動(表示線程)
        Thread t1 = new Thread(ft);
        t1.start();

        // 取得多線程結果
        Integer result = ft.get();
        System.out.println(result);
    }
}

import java.util.concurrent.Callable;

// Callable的泛型：結果的類型
// 與重寫方法call的返回值一致
public class MyCallable implements Callable<Integer> {
    @Override
    public Integer call() throws Exception {
        // 求1~100之間的和
        int sum = 0;
        for (int i = 0; i <= 100; i++) {
            sum = sum + i;
        }
        return sum;
    }
}

哪條線程執行到這方法，該線程就會在這裡停留對應的時間

# 6. 同步程式碼區域

把操作共享資料的程式碼鎖起來

synchronized() {
    操作共享資料的程式碼
}

# 特點：

。 鎖默認打開，有一個線程進去了，鎖自動關閉

。 裡面的程式碼全部執行完畢，線程出來，鎖自動打開

# 注意點：

。 要寫在循環內部，否則某個線程將會霸佔整個循環

。 使用時要建立鎖物件

。 鎖物件，一定要是唯一的。所有的線程都對應同一個鎖物件。一般會使用目前類的位元組碼檔案(class)，

public class ThreadDemo {
    public static void main(String[] args) {
        // 建立線程物件
        MyThread t1 = new MyThread();
        MyThread t2 = new MyThread();
        MyThread t3 = new MyThread();

        // 線程設定名字
        t1.setName("櫃台1");
        t2.setName("櫃台2");
        t3.setName("櫃台3");

        // 開啟線程
        t1.start();
        t2.start();
        t3.start();
    }
}

public class MyThread extends Thread{
    // 建立了多個物件
    // 加上static，這個類的所有物件都共享這個資料
    static int ticket = 0;

    // 鎖物件，一定要是唯一的
    static Object obj = new Object();

    @Override
    public void run() {
        while(true) {
            // 使用時要建立鎖物件
            synchronized (MyThread.class) { // 使用位元組碼檔案
                if(ticket < 100) {
                    try {
                        Thread.sleep(1000);
                    } catch (InterruptedException e) {
                        throw new RuntimeException(e);
                    }
                    ticket++;
                    System.out.println(getName() + "正在賣第" + ticket + "張票");
                }else{
                    break;
                }
            }
        }
    }
}

# 7. 同步方法

把synchronized關鍵字加到方法上

# 格式：

修飾符 synchronized 返回值類型 方法名(方法參數) {}

。 同步方法是所有方法裡面的所有程式碼

。 鎖物件不能自己指定

目前的方法非靜態：this()，目前方法的調用者

目前的方法是靜態：目前類的位元組碼檔案物件(class檔案)

public class ThreadDemo {
    public static void main(String[] args) {
        // 建立參數物件
        MyRunnable mr = new MyRunnable();

        // 建立線程物件
        Thread t1 = new Thread(mr);
        Thread t2 = new Thread(mr);
        Thread t3 = new Thread(mr);

        // 設定名字
        t1.setName("櫃台1");
        t2.setName("櫃台2");
        t3.setName("櫃台3");

        // 開啟線程
        t1.start();
        t2.start();
        t3.start();
    }
}

public class MyRunnable implements Runnable{

    // MyRunnable作為參數傳入線程物件，所以只會建立一次物件，不需要共享(不需要加上static)
    int ticket = 0;
    @Override
    public void run() {
        // 1. 循環
        while(true) {
            // 2. 同步程式碼區域(同步方法)
            synchronized (MyRunnable.class) {
                if (method()) break;

            }
        }
    }

    // 將同步程式碼區域抽出成一個獨立的方法
    // 加上synchronized
    private synchronized boolean method() {
        // 3. 判斷共享資料室是否到了末尾，如果到了末尾
        if(ticket == 100) {
            return true;
        }else{
            // 4. 判斷共享資料室是否到了末尾，如果沒有到了末尾
            try {
                Thread.sleep(10); //
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            }
            ticket++;
            System.out.println(Thread.currentThread().getName() + "在賣第" + ticket
             + "張票");
        }
        return false;
    }
}

# 8. lock鎖

JDK5以後提供了一個新的鎖物件Lock

Lock實現提供比使用synchronized方法和語句可以得到更廣泛的鎖定操作

Lock中提供了獲得鎖跟釋放鎖的方法

// 獲得鎖
void lock()

// 釋放鎖

void unlock();

Lock是接口，不能直接實體化，採用它的實現類ReentrantLock

ReentrantLock的構造方法

ReentrantLock()：使用空參構造

public class ThreadDemo {
    public static void main(String[] args) {
        // 建立參數物件
        MyRunnable mr = new MyRunnable();

        // 建立線程物件
        MyThread t1 = new MyThread();
        MyThread t2 = new MyThread();
        MyThread t3 = new MyThread();

        // 設定名字
        t1.setName("櫃台1");
        t2.setName("櫃台2");
        t3.setName("櫃台3");

        // 開啟線程
        t1.start();
        t2.start();
        t3.start();
    }
}

import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

public class MyThread extends Thread{
    // 加上static，這個類的所有物件都共享這個資料
    static int ticket = 0;

    // 使用繼承Thread類的方式進行實現多線程，會建立多個線程物件，資料要設定為static去共享
    static Lock lock = new ReentrantLock();

    @Override
    public void run() {
        while(true) {
            lock.lock();
            try {
                if(ticket == 100) {
                    break;
                }else{
                    Thread.sleep(10);
                    ticket++;
                    System.out.println(getName() + "正在賣第" + ticket + "張票");
                }
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            } finally { // 不論如何一定要將鎖打開
                lock.unlock();
            }
        }
    }
}

public class ThreadDemo {

public class MyThread extends Thread{

    static Object objB = new Object();

    @Override
    public void run() {
        // 1. 循環
        while(true) {
            if("線成A".equals((getName()))) {
                synchronized (objA) {
                    System.out.println("線程A拿到了A鎖，準備拿B鎖");
                    synchronized (objB) {
                        System.out.println("線程A拿到了A鎖，順利執行完一輪");;
                    }
                }
            }else if("線程B".equals(getName())) {
                if("線程B".equals(getName())) {
                    synchronized (objB) {
                        System.out.println("線程B拿到了B鎖，準備拿A鎖");
                        synchronized (objA) {
                            System.out.println("線程B拿到了B鎖，順利執行完一輪");
                        }
                    }
                }
            }
    }
}
}

# 10. 生產者 與 消費者(等待喚醒機制)

生產者(Producer)消費者(Consumer)模式是一個十分經典的多線程協作的模式

生產者：生產資料

消費者：消費資料

# 消費者：

  1. 判斷桌上是否有食物

  2. 如果沒有就等待(wait)

  3. 如果有就吃

  4. 吃完之後，喚醒廚師繼續做

# 生產者：

  1. 判斷桌上是否有食物

  2. 有：等待(wait)

  3. 沒有：製作食物

  4. 把食物放在桌子上

  5. 叫醒等待的消費者吃(notify)

# 1. 常見方法

# 2. 範例

# 1. 等待喚醒機制

public class ThreadDemo {
    public static void main(String[] args) {
        // 建立線程物件
        Cook c = new Cook(); 
        Foodie f = new Foodie();

        // 設定名字
        c.setName("廚師");
        f.setName("顧客");

        // 開啟線程
        c.start();
        f.start();
    }
}

public class Desk {
    // 用來控制生產者和消費者執行

    // 桌上是否有食物，0：沒有，1：有
    public static int foodFlog = 0;

    // 總個數，最多只到10
    public static int count = 10;

    // 鎖物件
    public static Object Lock = new Object();
}

public class Cook extends Thread{
    @Override
    public void run() {
        /*
         * 1.循環
         * 2.同步程式碼區域
         * 3.判斷共享資料是否到了末尾(到了末尾)
         * 4.判斷共享資料是否到了末尾(沒有到了末尾)
         */

        while (true) {
            synchronized (Desk.Lock) {
                if(Desk.count == 0) {
                    break;
                }else{
                    // 判斷桌子上是有否食物
                    if(Desk.foodFlog == 1) {
                        try {
                            // 如果有，等待
                            Desk.Lock.wait();
                        } catch (InterruptedException e) {
                            throw new RuntimeException(e);
                        }
                    }else{
                        // 如果沒有，製作食物
                        System.out.println("廚師製作了食物");
                        // 修改桌子狀態
                        Desk.foodFlog = 1;
                        // 喚醒消費者吃
                        Desk.Lock.notifyAll();
                    }
                }
            }
        }
    }
}

public class Foodie extends Thread{
    @Override
    public void run() {
        /*
         * 1.循環
         * 2.同步程式碼區域
         * 3.判斷共享資料是否到了末尾(到了末尾)
         * 4.判斷共享資料是否到了末尾(沒有到了末尾)
         */

        while (true) {
            synchronized (Desk.Lock) {
                if(Desk.count == 0) {
                    break;
                }else{
                    // 先判斷桌子上是否有食物
                    if(Desk.foodFlog == 0) {
                        // 沒有，等待
                        try {
                            Desk.Lock.wait(); // 讓目前線程跟鎖進行綁定
                        } catch (InterruptedException e) {
                            throw new RuntimeException(e);
                        }
                    }else {
                        // 把吃的總數-1
                        Desk.count--;

                        // 有，吃
                        System.out.println("正在吃東西，還能再吃"+ Desk.count + "次");

                        // 吃完後喚醒廚師繼續做
                        Desk.Lock.notifyAll();

                        // 修改桌子的狀態
                        Desk.foodFlog = 0;
                    }
                }
            }
        }
    }
}

import java.util.concurrent.ArrayBlockingQueue;

public class ThreadDemo {
    public static void main(String[] args) {
        // 生產者和消費者必須使用同一個阻塞隊列

        // 1. 建立阻塞對列的物件
        // 參數：長度
        ArrayBlockingQueue<String> queue = new ArrayBlockingQueue<>(1);

        // 2. 建立線程物件，並把阻塞對列傳遞過去
        Cook c = new Cook(queue);
        Foodie f = new Foodie(queue);
        
        // 3. 開啟線程
        c.start();
        f.start();
    }
}

import java.util.concurrent.ArrayBlockingQueue;

public class Cook extends Thread{

    // 只定義不給值
    ArrayBlockingQueue<String> queue;

    // 建立物件時才有值
    public Cook(ArrayBlockingQueue<String> queue) {
        this.queue = queue;
    }


    @Override
    public void run() {
        while (true) {
            // 不斷的把食物放到阻塞隊列中
            try {
                // put方法底層已經有鎖，不需要再加鎖
                queue.put("食物");
                System.out.println("廚師放了一個食物");
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            }
        }
    }
}


import java.util.concurrent.ArrayBlockingQueue;

public class Foodie extends Thread{

    // 只定義不給值
    ArrayBlockingQueue<String> queue;

    // 建立物件時才有值
    public Foodie(ArrayBlockingQueue<String> queue) {
        this.queue = queue;
    }

    @Override
    public void run() {
        while (true) {
            // 不斷的從阻塞隊列中取得食物
            try {
                // take方法底層也有鎖
                String food = queue.take();
                System.out.println(food);
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            }
        }
    }
}

# 2. 線程池主要核心原理

  1. 建立一個池子，池子中式空的

  2. 提交任務時，池子會建立新的線程物件，任務執行完畢，線程歸還池子。下次再次提交任務時，不需要建立新的線程，直接複用線有的線程即可

  3. 如果提交任務時，池子中沒有空閒線程，也無法建立新的線程，任務就會排隊等待

# 3. 線程池的工具類

Executors：線程池的工具類通過調用方法返回不同類型的線程物件

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class MyThreadPoolDemo {
    public static void main(String[] args) {

        // 1. 取得線程池物件
        ExecutorService pool1 = Executors.newCachedThreadPool();

        ExecutorService pool2 = Executors.newFixedThreadPool(3);

        // 2. 提交任務
        pool2.submit(new MyRunnable());

        // 3. 銷毀線程池
        pool1.shutdown();
    }
}

public class MyRunnable implements Runnable{
    @Override
    public void run() {
        for (int i = 1; i <= 100; i++) {
            System.out.println(Thread.currentThread().getName() + "---" + i);
        }
    }
}

import java.util.concurrent.ArrayBlockingQueue;
import java.util.concurrent.Executors;
import java.util.concurrent.ThreadPoolExecutor;
import java.util.concurrent.TimeUnit;

public class MyThreadPoolDemo1 {
    public static void main(String[] args) {
        ThreadPoolExecutor pool = new ThreadPoolExecutor(
                3, // 核心線程數量，不能小於0
                6, // 最大線程數量，不能小於0，最大數量 >= 核心線程數量，臨時線程：6 - 3 = 3
                60, // 空閒線程最大存活時間
                TimeUnit.SECONDS, // 時間單位
                new ArrayBlockingQueue<>(3), // 任務隊列
                Executors.defaultThreadFactory(), // 建立線程工廠
                new ThreadPoolExecutor.AbortPolicy() // 任務拒絕策略 // AbortPolicy是ThreadPoolExecutor的靜態內部類
        );
    }
}

# 自定義線程池總結

  1. 建立一個空的池子

  2. 有任務提交時，線程時會去建立線程去執行任務，執行完畢歸還線程

  3. 不斷提交的任務，會有以下三個臨界點

。當核心線程滿時，再提交任務就會排隊

。當核心線程滿時，隊伍滿時，會建立臨時線程

。當核心線程滿時，隊伍滿，臨時線程滿，會觸發任務拒絕策略
