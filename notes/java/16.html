# 16 【異常】

代表程式出現的問題

# 1. 介紹

# 1. Error

代表系統級別錯誤(屬於嚴重問題)

系統一旦出現問題，sun公司會把這些錯誤封裝成Error物件

Error是給sun公司自己使用的，不是給一般開發者使用的

# 2. Exception

Exception：異常，代表程式可能出現的問題。通常會用Exception以及它的子類來封裝成是出現的問題

執行時異常：RuntimeException及其子類，編譯階段不會出現異常提醒。執行時出現的異常(例如：陣列索引越界異常)

編譯時異常：編譯階段濟會出現異常提醒(例如：日期解析異常

# 3. 總結

  1. 異常是程式中可能出現的問題

  2. 最上層父類：Exception。分為兩類：執行時異常、編譯時異常

  3. 編譯時異常：沒有繼承RuntimeException的異常，直接繼承於Exception，編譯階段就會出現錯誤提醒

  4. 編譯時異常：RuntimeException本身和子類。編譯階段沒有錯誤提醒，在執行時出現

import java.text.ParseException;
import java.text.SimpleDateFormat;
import java.util.Date;

public class Exception {
    public static void main(String[] args) throws ParseException {
        // 編譯時異常
        // 在編譯階段，必須手動處理，否則出現錯誤
        String time = "2030年1月1日";
        SimpleDateFormat sdf = new SimpleDateFormat("yyyy年MM月dd日");
        Date date = sdf.parse(time); // throws ParseException
        System.out.println(date);

        // 執行時異常
        // 編譯階段不需要處理，執行時出現的
        int[] arr = {1,2,3};
        System.out.println(arr[3]);
    }
}

# 2. 作用

# 作用一：用來查詢bug的關鍵參考訊息

範例一：

public class Exception1 {
    public static void main(String[] args) {
        // 空指針異常：NullPointerException
        Student[] arr = new Student[3]; // 沒有建立物件將資料放入，所以出現異常
        String name = arr[0].getName();
        System.out.println(name);
    }
}

public class Student {
    private String name;
    private int age;

    public Student() {
    }

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    @Override
    public String toString() {
        return "Student{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }
}

範例二：

public class Exception2 {
    public static void main(String[] args) {
        Student2 stu = new Student2("S1", 20);

                '}';
    }
}

public class Exception3 {

    }

public class Student {

    }

# 3. 異常處理方式

# 1. JVM默認的處理方式

把異常的名稱，異常原因以及異常出現的位置等訊息輸出在控制台

將程式停止執行，異常處之後的程式碼不會再執行

public class Exception5 {
    public static void main(String[] args) {
        System.out.println("A");
        System.out.println(2/0); // 算術異常
        System.out.println("B");
        System.out.println("C");
    }
}

# 2. 自行處理(捕獲異常)

目的：帶程式碼出現異常時，可以讓程式繼續往下執行

try{
    可能出現的異常的程式碼
}catch{
    異常的處理程式碼
}

public class Exception5 {
    public static void main(String[] args) {
        int[] arr = {1,2,3,4,5};
        
        try{
            System.out.println(arr[10]);
            // 此處出現了異常，程式就會在這裡建立一個ArrayIndexOutOfBoundsException物件
            // new ArrayIndexOutOfBoundsException
            // 拿著這個物件到catch的()中對比，看括號中的變數是否能接收這個物件
            // 如果能被接收，就表示異常被捕獲，執行catch裡面對應的程式碼
            // 當catch裡面所有的程式碼執行完畢，繼續執行try...catch下面的其他程式碼
        }catch(ArrayIndexOutOfBoundsException e){
            System.out.println("索引越界了");
        }
        System.out.println("繼續執行");
    }
}

# 疑問

  1. 如果try中沒有遇到問題，怎麼執行

會把trya裡面所有的程式碼全部執行完畢，不會執行catch的程式碼

只有當異常出現才會執行catch的程式碼

  1. 如果try中可能會遇到多個問題，怎麼執行

會寫多個catch與之對應

如果這些異常存在父子關係的話，父類一定要寫在下面

在JDK以後，可以在catch中同時捕獲多個異常，中間用|分隔

// 如果出現A異常或B異常，採取同一種方案
catch((ArrayIndexOutOfBoundsException | ArithmeticException e))
  1. 如果try中遇到的問題沒有被捕獲，怎麼執行

會交給虛擬機進行處理(控制台輸出錯誤)，並結束程式

  1. 如果try中遇到了問題，那麼try下面的程式碼還會被執行嗎

下面的程式碼就不會執行了，直接跳轉到對應的catch當中，執行catch裡面的語句

如果沒有對應的catch，則交給虛擬機處裡

# 3. 異常的中常見方法

方法名稱說明
public String getMessage()返回此throwable的詳細訊息字串
public String toString()返回此可拋出的簡短敘述
public void printStackTrace()把異常的錯誤輸出在控制台
public class Exception5 {
    public static void main(String[] args) {
        int[] arr = {1,2,3,4,5,6};
        try {
            System.out.println(arr[10]);
        } catch (ArrayIndexOutOfBoundsException e) {
            String message = e.getMessage(); // Index 10 out of bounds for length 6
            System.out.println(message);

            String str = e.toString();
            System.out.println(str); // java.lang.ArrayIndexOutOfBoundsException: Index 10 out of bounds for length 6

            // 不會結束虛擬機，不會停止程式執行
            // 包含上面兩個結果
            e.printStackTrace();
        }
    }
}

# 3. 拋出異常(交由調用者處理)

  1. throws

寫在方法定義處，表示宣告一個異常，告訴調用者，使用本方法可能會有那些異常

public void 方法()throws 異常類名1, 異常類名2...{

}

。編譯時異常：必須要寫

。執行時異常：可以不寫

  1. throw

寫在方法內，表示結束方法

手動拋出異常物件，交給調用者

方法中下面的程式碼不在執行了

public void 方法(){
    throw new NullPointerExecption();   
}

public class Exception5 {
    public static void main(String[] args) {
        int[] arr = {1,2,3,4,5,6};
        int max = 0;
        try {
            max = getMax(arr);
        } catch (NullPointerException e) { // 如果沒有捕獲就會交由虛擬機處理
            e.printStackTrace();
            System.out.println("空指針異常");
        } catch (ArrayIndexOutOfBoundsException e) {
            e.printStackTrace();
            System.out.println("索引越界異常");
        }
        System.out.println(max);

    }


//    public static int getMax(int[] arr) throws NullPointerException, ArrayIndexOutOfBoundsException { // RuntimeException的子類，屬於執行時異常，不需要手動宣告

    public static int getMax(int[] arr) throws NullPointerException, ArrayIndexOutOfBoundsException { // RuntimeException的子類，屬於執行時異常，不需要手動宣告
        if(arr == null) {
            // 手動建立一個異常物件，並把這個異常交給方法的調用者處理
            // 方法結束，下面的程式不會再執行，程式結束
            throw new NullPointerException();
        }

        if(arr.length == 0) {
            // 手動建立一個異常物件，並把這個異常交給方法的調用者處理
            // 方法結束，下面的程式不會再執行，程式結束
            throw new ArrayIndexOutOfBoundsException();
        }

        int max = arr[0];
        for (int i = 0; i < arr.length; i++) {
            if(arr[i] > arr[max]) {
                max = arr[i];
            }
        }
        return max;
    }
}

# 4. 總結

  1. 虛擬機默認處理異常的方式

把異常訊息以紅色字體輸出在控制台，並結束程式

  1. 捕獲try...catch

一般用於調用處，能讓程式碼繼續往下執行

  1. 拋出throw throws

在方法中，出現異常了，方法就沒有繼續執行下去的意義了，此時採取拋出處理，讓該方法結束執行並告訴調用者出現了問題
